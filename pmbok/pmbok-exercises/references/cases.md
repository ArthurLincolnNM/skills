## CASE001: Projeto de Software ERP
**Cenário:** Você é PM de um projeto de implementação de ERP com budget=$500,000 (BAC) e prazo=12 meses. Estamos no mês 6.
- EV = $220,000 (44% do trabalho feito)
- PV planejado para o mês 6 = $250,000 (50%)
- AC = $280,000
- Equipe de 6 pessoas + você + sponsor
- Um membro da equipe tem conflitos frequentes com outros
- Um fornecedor de hardware está pedindo mudança de preço (+20%)

### P1: Qual é o CPI? O projeto está over ou under budget?
**A)** CPI = 0.786 → over budget
**B)** CPI = 0.917 → over budget
**C)** CPI = 1.27 → under budget
**D)** CPI = 1.0 → on budget

**Correct:** A — CPI = EV/AC = $220,000/$280,000 = 0.786 (< 1 → over budget)

### P2: Qual é o SPI? O projeto está ahead ou behind schedule?
**A)** SPI = 0.88 → behind schedule
**B)** SPI = 1.1 → ahead schedule
**C)** SPI = 0.75 → behind schedule
**D)** SPI = 1.0 → on schedule

**Correct:** A — SPI = EV/PV = $220,000/$250,000 = 0.88 (< 1 → behind schedule)

### P3: Qual é o EAC mais provável?
**A)** $636,297 (EAC = BAC/CPI = 500000/0.786 ≈ 636,297)
**B)** $600,000
**C)** $500,000
**D)** $700,000

**Correct:** A — EAC = BAC/CPI = $636,297. Projeto vai custar ~$136k a mais que o planejado.

### P4: Quantas canais de comunicação existem na equipe?
**A)** 8
**B)** 28 (n = 8: 8×7/2 = 28)
**C)** 56
**D)** 36

**Correct:** B — 8 pessoas (6 team + PM + sponsor): 8×7/2 = 28 canais. Communication overhead = problema.

### P5: O que você deve fazer PRIMEIRO sobre o conflito na equipe?
**A)** Fire the team member
**B)** Confrontar/dialogar diretamente (collab de primeira linha)
**C)** Reportar ao RH
**D)** Ignorar e esperar resolver sozinho

**Correct:** B — PMI approach: direct confrontation/problem solving é a primeira linha para conflitos importantes.

### P6: Sobre a mudança do fornecedor de hardware: o que você deve fazer PRIMEIRO?
**A)** Recusar sumariamente
**B)** Implementar a mudança para manter o fornecedor feliz
**C)** Realizar Perform Integrated Change Control → evaluate impact
**D)** Pedir ao sponsor que decida

**Correct:** C — TODAS mudanças passam pelo Perform Integrated Change Control primeiro. O PM não decide sozinho nem delega sem análise.
