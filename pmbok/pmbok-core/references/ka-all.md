# PMBOK 6th — Knowledge Areas (All 10)
## 1. Project Integration Management (7 processes)
The art of coordinating all project elements. The PM is always accountable.
- **4.1 Develop Project Charter** (Initiating) — authorizes the project
- **4.2 Develop Project Management Plan** (Planning) — consolidating plan
- **4.3 Direct and Manage Project Work** (Executing) — doing the work
- **4.4 Manage Project Knowledge** (Executing) — organizational knowledge
- **4.5 Monitor and Control Project Work** (M&C) — tracking performance
- **4.6 Perform Integrated Change Control** (M&C) — reviewing changes
- **4.7 Close Project or Phase** (Closing) — formal completion
### Key Concept: The Project Management Plan is the MAIN output of planning. All other management plans are subsidiaries.
### Key Concept: "Perform Integrated Change Control" = evaluating ALL proposed changes (not just scope).
---
## 2. Project Scope Management (6 processes)
Ensuring the project includes all the work required — and ONLY that work.
- **5.1 Plan Scope Management** (Planning)
- **5.2 Collect Requirements** (Planning) — eliciting needs
- **5.3 Define Scope** (Planning) — scope statement
- **5.4 Create WBS** (Planning) — work breakdown structure
- **5.5 Validate Scope** (Executing) — formal acceptance
- **5.6 Control Scope** (M&C) — monitoring scope changes
### Key Tools: Requirements Documentation, Scope Statement, WBS, WBS Dictionary
### WBS = hierarchical decomposition of total scope. 100% rule: includes all work.
### Scope vs Requirements: Scope = what the project will deliver. Requirements = features/functions needed.
---
## 3. Project Schedule Management (6 processes)
Ensuring timely completion of the project.
- **6.1 Plan Schedule Management** (Planning)
- **6.2 Define Activities** (Planning)
- **6.3 Sequence Activities** (Planning) — identifying dependencies
- **6.4 Estimate Activity Durations** (Planning) — estimating how long each activity
- **6.5 Develop Schedule** (Planning) — critical path, network analysis
- **6.6 Control Schedule** (M&C) — monitoring progress
### Key Tools: Activity List, Network Diagram, Critical Path Method, PERT, Schedule Network Analysis
### Critical Path = longest path through network diagram. Activities on CP have ZERO float.
### Float (Slack) = amount of time an activity can be delayed without delaying the project.
### Total Float = LS - ES. Free Float = amount a successor can be delayed.
---
## 4. Project Cost Management (4 processes)
Ensuring the project is completed within the approved budget.
- **7.1 Plan Cost Management** (Planning)
- **7.2 Estimate Costs** (Planning) — cost of resources
- **7.3 Determine Budget** (Planning) — aggregating to cost baseline
- **7.4 Control Costs** (M&C) — monitoring and controlling cost
### Key Tools: EVM (Earned Value Management), Cost Baseline, Cost Management Plan
### EVM integrates scope, schedule, and cost performance.
### Key Formulas:
- BAC = Budget at Completion
- CPI = EV / AC → cost performance (CPI < 1 = over budget)
- SPI = EV / PV → schedule performance (SPI < 1 = behind schedule)
- EAC = BAC / CPI (most common) or EAC = AC + (BAC - EV) / CPI
- VAC = BAC - EAC
### Earned Value (EV) = % complete × BAC. EV is in dollars (or hours for FP projects).
---
## 5. Project Quality Management (4 processes)
Ensuring the project will satisfy the needs that precipitated it.
- **8.1 Plan Quality Management** (Planning) — quality standards, metrics
- **8.2 Manage Quality** (Executing) — performing quality activities
- **8.3 Control Quality** (M&C) — validating deliverables meet requirements
### Key Concepts: Cost of Quality (COQ) = Cost of Conformance + Cost of Nonconformance
### Tools: Seven Basic Tools (affinity, tree, process flow, control charts, Pareto, histograms, scatter, matrix)
### QA vs QC: Quality Assurance = process improvement. Quality Control = output validation.
---
## 6. Project Resource Management (7 processes)
Identifying, acquiring, developing, and managing resources.
- **9.1 Plan Resource Management** (Planning) — roles, responsibilities
- **9.2 Estimate Activity Resources** (Planning) — resource types and quantities
- **9.3 Acquire Resources** (Executing) — getting team and physical resources
- **9.4 Develop Team** (Executing) — improving competencies, teamwork
- **9.5 Manage Team** (Executing) — tracking performance, feedback
- **9.6 Control Resources** (M&C) — monitoring resource allocation
### Key Concepts: RAM (RACI matrix), resource leveling,Tuckman team development stages
### Tuckman: Forming → Storming → Norming → Performing
### Maslow: Physiological → Safety → Social → Esteem → Self-actualization
### Theory X: top-down control. Theory Y: self-direction. Theory Z: collective focus.
---
## 7. Project Communications Management (5 processes)
Ensuring timely and appropriate collection, creation, distribution, storage, retrieval, and disposal of project information.
- **10.1 Plan Communications Management** (Planning) — communication needs, methods
- **10.2 Manage Communications** (Executing) — creating and distributing information
- **10.3 Monitor Communications** (M&C) — ensuring communication effectiveness
### Key Concepts: Communication Model, Communication Channels = n × (n-1) / 2
### Push (email) vs Pull (intranet) vs Interactive (meetings)
### Key Formula: Communication Channels = n(n-1)/2 where n = number of stakeholders
### Communication Management Plan = WHEN, WHAT, WHO, HOW, FEEDBACK. Always includes escalation paths.
---
## 8. Project Risk Management (7 processes)
Planning how to deal with uncertainty.
- **11.1 Plan Risk Management** (Planning) — how to identify, analyze, respond
- **11.2 Identify Risks** (Planning) — documenting risk events
- **11.3 Perform Qualitative Risk Analysis** (Planning) — probability × impact matrix
- **11.4 Perform Quantitative Risk Analysis** (Planning) — modeling, simulation
- **11.5 Plan Risk Responses** (Planning) — strategies for threats and opportunities
- **11.6 Implement Risk Responses** (Executing) — executing risk response plan
- **11.7 Monitor Risks** (M&C) — tracking identified risks, residual risks, new risks
### Threat Strategies: Avoid, Transmit (transfer), Mitigate, Accept
### Opportunity Strategies: Exploit, Share, Enhance, Accept
### Contingency Plans vs Fallback Plans vs Workarounds (not pre-planned)
### Expected Monetary Value (EMV) = Probability × Impact
### Monte Carlo = simulation to determine probability of meeting objectives
---
## 9. Project Procurement Management (4 processes)
Acquiring products, services, or results from outside the project team.
- **12.1 Plan Procurement Management** (Planning) — procurement strategy
- **12.2 Conduct Procurements** (Executing) — getting seller responses
- **12.3 Control Procurements** (M&C) — managing procurement relationships
- **12.4 Close Procurements** (Closing) — finalizing procurement
### Key Concepts: Fixed Price (FP) vs Cost-Reimbursable (CR) vs Time & Materials (T&M)
### Contract Types:
- FFP: Firm Fixed Price (seller bears risk)
- FP-EPA: Fixed Price with Economic Price Adjustment
- CR: Cost Plus Percentage of Cost (obsoleto)
- CPFF: Cost Plus Fixed Fee
- CPIF: Cost Plus Incentive Fee
- T&M: hybrid (ceiling rate)
### Make or Buy Analysis: decision to procure or internal perform
---
## 10. Project Stakeholder Management (4 processes)
Identifying, analyzing, and engaging stakeholders.
- **13.1 Identify Stakeholders** (Initiating) — identifying all stakeholders
- **13.2 Plan Stakeholder Engagement** (Planning) — strategies for engagement
- **13.3 Manage Stakeholder Engagement** (Executing) — engaging stakeholders
- **13.4 Monitor Stakeholder Engagement** (M&C) — monitoring engagement
### Key Concepts: Power/Interest Grid, Stakeholder Register, Engagement Assessment Matrix
### Grid: Power-Interest (or Salience) model for prioritization
### Stakeholder engagement assessment: C = Current, D = Desired, M = Management strategy
