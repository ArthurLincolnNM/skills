# PMBOK 6th — Decision Frameworks
## "What should you do FIRST?"
1. **Is there a project?** → No → Escalate / Get authorization
2. **Is there a Project Charter?** → No → Develop Project Charter (4.1)
3. **Is there a process for that?** → No → Create a plan / Tailor the approach
4. **Is this a change request?** → Yes → Perform Integrated Change Control (4.5)
5. **Is this a stakeholder issue?** → Identify Stakeholder → Plan Engagement → Manage
## "What is the BEST answer?"
**Principled approach (PMI mindset):**
1. **Professional and Social Responsibility** — always the override answer
2. **Stakeholder satisfaction** — the ultimate project success measure
3. **Follow the process first** — PMBOK is a process framework
4. **PM is accountable** — even if not directly responsible
## "Which process handles this?"
| Situation | Process | KA | PG |
|-----------|---------|----|----|
| "The team doesn't understand the plan" | Manage Communications | Comms | EXEC |
| "Scope is changing" | Control Scope | Scope | M&C |
| "A vendor is late" | Control Procurements | Procure | M&C |
| "Team member conflicts" | Manage Team | Resource | EXEC |
| "Schedule is slipping" | Control Schedule | Schedule | M&C |
| "Budget is over" | Control Costs | Cost | M&C |
| "New risk identified" | Identify Risks + Plan Risk Responses | Risk | PLAN |
| "Stakeholder not engaged" | Manage Stakeholder Engagement | Stakeholder | EXEC |
| "Quality issue found" | Control Quality | Quality | M&C |
| "Lessons learned" | Manage Project Knowledge | Integration | EXEC |
## "What type of contract?"
| Situation | Contract Type |
|-----------|------------|
| Well-defined scope, fixed deliverables | FFP |
| Exploratory R&D, unknown scope | CPFF (Cost Plus Fixed Fee) |
| Time and materials, indefinite scope | T&M |
| Low risk, well-defined | FP-EPA |
| High risk, uncertain | CPIF |
| Long-term, economic changes | FP-EPA |
## "Which conflict resolution?"
| Strategy | When to Use |
|-----------|-----------|
| **Confronting / Problem Solving** | Time to solve it, both parties reasonable |
| **Compromising / Realignment** | Time pressure, temporary solution |
| **Smoothing / Accommodation** | Relationship matters more than the issue |
| **Forcing / Directing** | Emergency, one-time decision, you have authority |
| **Withdrawing / Avoiding** | Issue is trivial, no one will win |
| **Collaborating / Problem Solving** | Long-term relationship, important issue |
## "Risk strategy?"
| Type | Strategies |
|------|---------|
| **Threat (negative risk)** | Avoid, Transfer, Mitigate, Accept |
| **Opportunity (positive risk)** | Exploit, Share, Enhance, Accept |
| **Residual risk** | Accept (after primary response) |
| **Secondary risk** | Plan a response for the secondary risk too |

| Brainstorming | Generating many ideas |

## "What is the issue?"
| Symptom | Root Cause | Process |
|---------|-----------|---------|
| Budget over | CPI < 1 | Control Costs |
| Schedule delay | SPI < 1 | Control Schedule |
| Quality defect | No quality standards | Control Quality |
| Stakeholder unhappy | Not engaged | Manage Stakeholder Engagement |
| Team conflict | No clear roles | Manage Team |
| Change explosion | Scope baseline not set | Control Scope |
| Communication failure | No plan | Plan Communications |
